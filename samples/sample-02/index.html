HO8 · Sample 02

Customer Lookup MCP Server

Expose a lookup_customer tool that lets Claude query a local in-memory customer database by ID or email.

Week 4 M12 · MCP Solo Database
Uses your Claude.ai subscription via Claude Desktop — no ANTHROPIC_API_KEY needed.

Claude Desktop connects to this MCP server locally; your subscription provides the AI.

What this MCP server does

This server registers one tool called lookup_customer. When you ask Claude something like "what plan is alice@acmecorp.com on?", Claude calls the tool, which looks up the customer from a hard-coded in-memory dictionary and returns their name, plan tier, account creation date, and open ticket count.

The database lives entirely in server.py — no external database or API key is required.

Requirements
  • Python 3.10 or newer
  • Claude Desktop — free download at claude.ai/download (uses your Claude.ai subscription, no API key)
  • mcp Python package (installed below)
Build the server

Open a terminal and run:

# 1. Go into the sample directory
cd samples/sample-02

# 2. (Recommended) Create a virtual environment
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows PowerShell

# 3. Install dependencies
pip install -r requirements.txt

# 4. Verify the server starts
python server.py

You should see: Starting Customer Lookup MCP server.... Press Ctrl+C to stop — Claude Desktop will launch it automatically when needed.

Connect to Claude Desktop

Open Claude Desktop → Settings → Developer → Edit Config and paste the following into claude_desktop_config.json. Replace the path with the absolute path to server.py on your machine.

{
  "mcpServers": {
    "customer-lookup": {
      "command": "python",
      "args": ["/ABSOLUTE/PATH/TO/samples/sample-02/server.py"],
      "env": {}
    }
  }
}

Tips:

  • On macOS the path looks like /Users/yourname/ho8-mcp-server/samples/sample-02/server.py
  • On Windows use the full path with backslashes, e.g. C:\Users\yourname\ho8-mcp-server\samples\sample-02\server.py
  • If you used a virtual environment, replace python with the full path to the venv Python, e.g. /Users/yourname/ho8-mcp-server/samples/sample-02/.venv/bin/python
  • Save the file and restart Claude Desktop to load the new server.
Test it

After restarting Claude Desktop, type any of these prompts in a new conversation:

"Look up customer C001"
"What plan is alice@acmecorp.com on?"
"How many open tickets does Carol White have?"
"List all enterprise customers"

You should see Claude call the lookup_customer tool (visible as a tool-use block) and return customer details. If the tool icon appears in the chat toolbar, the server is connected correctly.
