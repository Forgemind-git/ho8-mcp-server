HO8 · Sample 03

Task Manager MCP Server

Expose a manage_tasks tool that lets Claude list, add, and complete tasks stored in a local tasks.json file.

Week 4 M12 · MCP Solo Read + Write
Uses your Claude.ai subscription via Claude Desktop — no ANTHROPIC_API_KEY needed.

Claude Desktop connects to this MCP server locally; your subscription provides the AI.

What this MCP server does

This server provides a single MCP tool called manage_tasks with three actions:

  • list — show all tasks with their status (TODO / DONE)
  • add — create a new task with a description
  • complete — mark a task done by its ID number or a title substring

Tasks are persisted to a tasks.json file in the same directory as server.py, so they survive across Claude Desktop restarts. No database or external service is needed.

Requirements
  • Python 3.10 or newer
  • Claude Desktop — free download at claude.ai/download (uses your Claude.ai subscription, no API key)
  • mcp Python package (installed below)
Build the server

Open a terminal and run:

# 1. Go into the sample directory
cd samples/sample-03

# 2. (Recommended) Create a virtual environment
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows PowerShell

# 3. Install dependencies
pip install -r requirements.txt

# 4. Verify the server starts
python server.py

You should see output like:

Starting Task Manager MCP server...
Tasks file: /path/to/samples/sample-03/tasks.json

Press Ctrl+C to stop — Claude Desktop will launch it automatically when needed.

Connect to Claude Desktop

Open Claude Desktop → Settings → Developer → Edit Config and paste the following into claude_desktop_config.json. Replace the path with the absolute path to server.py on your machine.

{
  "mcpServers": {
    "task-manager": {
      "command": "python",
      "args": ["/ABSOLUTE/PATH/TO/samples/sample-03/server.py"],
      "env": {}
    }
  }
}

Tips:

  • On macOS the path looks like /Users/yourname/ho8-mcp-server/samples/sample-03/server.py
  • On Windows use the full path with backslashes, e.g. C:\Users\yourname\ho8-mcp-server\samples\sample-03\server.py
  • If you used a virtual environment, replace python with the full path to the venv Python, e.g. /Users/yourname/ho8-mcp-server/samples/sample-03/.venv/bin/python
  • Save the file and restart Claude Desktop to load the new server.
Test it

After restarting Claude Desktop, type any of these prompts in a new conversation:

"Add a task: review pull requests"
"What tasks do I have?"
"Mark the 'review pull requests' task as done"
"Add three tasks: write tests, update docs, and deploy to staging"

You should see Claude call the manage_tasks tool (visible as a tool-use block) and confirm each action. Check the tasks.json file in samples/sample-03/ to see tasks being persisted. If the tool icon appears in the chat toolbar, the server is connected correctly.
