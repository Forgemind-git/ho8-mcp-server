HO8 · Sample 05

Document Search MCP Server

Expose a search_docs tool that performs keyword search across a folder of .txt and .md files, returning excerpts with context.

Week 4 M12 · MCP Solo Search
Uses your Claude.ai subscription via Claude Desktop — no ANTHROPIC_API_KEY needed.

Claude Desktop connects to this MCP server locally; your subscription provides the AI.

What this MCP server does

This server exposes a single tool called search_docs. Give it a keyword or phrase and it scans all .txt and .md files in a specified folder (defaulting to the docs/ subfolder), returning up to 10 matching excerpts with 3 lines of surrounding context.

The sample comes with four pre-built documents in docs/:

  • getting-started.md — product onboarding guide
  • billing.md — subscription and billing FAQ
  • api-reference.txt — API endpoint reference
  • faq.txt — general frequently asked questions

No external service or index is required — the server reads files directly from disk.

Requirements
  • Python 3.10 or newer
  • Claude Desktop — free download at claude.ai/download (uses your Claude.ai subscription, no API key)
  • mcp Python package (installed below)
Build the server

Open a terminal and run:

# 1. Go into the sample directory
cd samples/sample-05

# 2. (Recommended) Create a virtual environment
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows PowerShell

# 3. Install dependencies
pip install -r requirements.txt

# 4. Verify the server starts
python server.py

You should see:

Starting Document Search MCP server...
Default docs folder: /path/to/samples/sample-05/docs

Press Ctrl+C to stop — Claude Desktop will launch it automatically when needed.

Want to search your own documents? Drop any .txt or .md files into the docs/ folder (or pass a custom folder path to the tool) and the server will search them automatically.

Connect to Claude Desktop

Open Claude Desktop → Settings → Developer → Edit Config and paste the following into claude_desktop_config.json. Replace the path with the absolute path to server.py on your machine.

{
  "mcpServers": {
    "doc-search": {
      "command": "python",
      "args": ["/ABSOLUTE/PATH/TO/samples/sample-05/server.py"],
      "env": {}
    }
  }
}

Tips:

  • On macOS the path looks like /Users/yourname/ho8-mcp-server/samples/sample-05/server.py
  • On Windows use the full path with backslashes, e.g. C:\Users\yourname\ho8-mcp-server\samples\sample-05\server.py
  • If you used a virtual environment, replace python with the full path to the venv Python, e.g. /Users/yourname/ho8-mcp-server/samples/sample-05/.venv/bin/python
  • Save the file and restart Claude Desktop to load the new server.
Test it

After restarting Claude Desktop, type any of these prompts in a new conversation:

"Search the docs for 'billing'"
"What does the documentation say about API rate limits?"
"Find anything in the docs mentioning 'refund'"
"Search for 'getting started' in the docs and summarize what you find"

You should see Claude call the search_docs tool (visible as a tool-use block) and return matching excerpts from the docs/ folder. If the tool icon appears in the chat toolbar, the server is connected correctly.
